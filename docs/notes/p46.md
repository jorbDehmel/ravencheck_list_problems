# P46

The problem is as below.

```lisp
(declare-sort Any 0)
(declare-datatype
  list (par (a) ((nil) (cons (head a) (tail (list a))))))
(define-fun-rec
  weird_concat
  (par (a) (((x (list (list a)))) (list a)))
  (match x
    ((nil (_ nil a))
     ((cons y xss)
      (match y
        ((nil (weird_concat xss))
         ((cons z xs) (cons z (weird_concat (cons xs xss))))))))))
(define-fun-rec
  ++
  (par (a) (((x (list a)) (y (list a))) (list a)))
  (match x
    ((nil y)
     ((cons z xs) (cons z (++ xs y))))))
(define-fun-rec
  concat
  (par (a) (((x (list (list a)))) (list a)))
  (match x
    ((nil (_ nil a))
     ((cons y xs) (++ y (concat xs))))))
(prove
  (par (a)
    (forall ((x (list (list Any)))) (= (concat x) (weird_concat x)))))
```

The following is the initial implementation of the problem.

```rust
#[ravencheck::check_module]
#[allow(dead_code)]
mod p46 {
  // Note: I'm just going to use A == Any for generic A. I don't
  // know why they haven't done that in the spec.

  #[define]
  #[derive(PartialEq, Clone)]
  pub enum LinkedList<T> {
    Nil,
    Cons(T, Box<LinkedList<T>>),
  }

  #[define]
  #[recursive]
  pub fn append<T>(x: LinkedList<T>, y: LinkedList<T>) -> LinkedList<T> {
    match x {
      LinkedList::<T>::Nil => y,
      LinkedList::<T>::Cons(z, xs) => LinkedList::<T>::Cons(z, Box::new(append::<T>(*xs, y)))
    }
  }

  #[define]
  #[recursive]
  fn weird_concat<A>(x: LinkedList<LinkedList<A>>) -> LinkedList<A> {
    match x {
      LinkedList::<LinkedList<A>>::Nil => LinkedList::<A>::Nil,
      LinkedList::<LinkedList<A>>::Cons(y, xss) => match y {
        LinkedList::<A>::Nil => weird_concat::<A>(*xss),
        LinkedList::<A>::Cons(z, xs) => LinkedList::<A>::Cons(
          z,
          Box::new(
            weird_concat::<A>(
              LinkedList::<LinkedList<A>>::Cons(
                *xs,
                Box::new(*xss)
              )
            )
          )
        )
      }
    }
  }

  #[define]
  #[recursive]
  fn normal_concat<A>(x: LinkedList<LinkedList<A>>) -> LinkedList<A> {
    match x {
      LinkedList::<LinkedList<A>>::Nil => LinkedList::<A>::Nil,
      LinkedList::<LinkedList<A>>::Cons(y, xs) =>
        append::<A>(
          y,
          normal_concat::<A>(*xs)
        )
    }
  }

  #[annotate]
  #[for_type(LinkedList<A> => <A>)]
  #[inductive(x: LinkedList<LinkedList<A>>)]
  fn p46<A>() -> bool {
    normal_concat::<A>(x) == weird_concat::<A>(x)
  }
}

```

## Lemma 3

Now we want to find a lemma that implies the Cons->Cons->Cons
case.

```rust
// Assume
normal_concat(x) == weird_concat(x)

// WTS forall a
normal_concat(Cons(a, x)) == weird_concat(Cons(a, x))

// Definition
append(a, normal_concat(x)) == match a {
  Nil => weird_concat(x),
  Cons(z, xs) => Cons(z, weird_concat(Cons(xs, x)))
}

// If a == Nil
normal_concat(x) == weird_concat(x)
// IH

// If a == Cons(z, xs)

// Lemma 3
append(Cons(z, xs), normal_concat(x))
==
Cons(z, weird_concat(Cons(xs, x)))
```

This final statement is sufficient to imply `p46`, so we will
make it `lemma_3`.

```rust

    z,
    weird_concat::<A>(
      LinkedList::<LinkedList<A>>::Cons(xs, x)

  )
}
```

## Revisiting lemma 3 like a month later

From Nick: "Case [cons->cons->cons] means that, when unrolled,
we are in the `cons` branch, then the `cons` branch, then the
`cons` branch of the VC: It's not necessarily the same as the
inductive instance, but frequently is".

Let's unroll the VC of lemma 3 and find `Nil->Cons->Cons->Nil`.

```rust
match xs {

      z_prime,
```

That arrives back at p46: Maybe this isn't a useful lemma.
